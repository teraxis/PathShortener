## v1.9.1 — читабельні скорочення (поточний)

- **Кінець «Дтр Грн Нзл»**: механічне викидання голосних замінено на читабельне
  обрізання слів до впізнаваного кореня (`Compaction.ClipWords`):
  «незалежності»→«незале.», «гарантій»→«гарант.», «Дотримання»→«Дотрим.».
- **Розширено словник** службових і юридично-адміністративних слів (гарантії,
  незалежність, втручання, звернення, повідомлення, комісія, секретаріат тощо) —
  звичайний аналіз одразу дає зрозумілі назви.
- **Виправлено регістр акронімів**: ВККСУ/ВРП/СБУ більше не перетворюються на
  «Вкксу»/«Врп» (правила зберігають великі літери акронімів).
- **Переписано промпт ШІ** на людино-орієнтований: залишати 1–3 найважливіші
  слова цілими, прибирати службові слова, використовувати сталі абревіатури й
  **ніколи не викидати голосні**; додано приклади «як треба / як не можна».

## v1.9.0 — перебудова інтерфейсу, статистика

- **Вкладки зліва, на всю ширину**: замість правого сайдбару — вертикальна
  стрічка вкладок ліворуч. Вкладки: **Аналіз** (кнопки аналізу + таблиця плану),
  **Папки**, **Архіви**, **Словник** (окрема вкладка), **Налаштування**,
  **Статистика**. Кнопки «Проаналізувати» тощо тепер усередині вкладки «Аналіз».
- **Горизонтальний скрол у таблицях** шляхів завжди видимий; стовпці з фіксованою
  шириною можна розширювати — фрейм розтягується й з'являється прокрутка.
- **Вкладка «Статистика»**: за останній аналіз показує кількість джерел, шляхів,
  назву моделі (якщо використовувалась), кількість викликів моделі (пакетів),
  приблизну кількість токенів і скорочень прийнято/відхилено.
- **Кнопка «Стоп» тепер справді перериває виконання**: у застосувач
  (розпакування, перейменування, копіювання) додано перевірку скасування між
  елементами — раніше `Task.Run` продовжував роботу після натискання.
- **Оновлено списки моделей** у профілях: Claude (Opus 4.8 / Sonnet 5 / Haiku 4.5 /
  Fable 5), OpenAI (gpt-4.1 / 4o / o4-mini / o3), Grok (grok-4/3), Gemini (2.5).
- Аналіз і застосування розділено остаточно: «Застосувати» виконує поточний
  (можливо, відредагований) план.

## v1.8.0 — таблиця, словник, конфлікти

- **Ручне редагування** нових шляхів прямо в таблиці плану; відредаговані рядки
  позначаються джерелом «ручне». Аналіз і застосування розділено: «Застосувати»
  виконує поточний (можливо, змінений вручну) план, а не переобраховує його.
- **Каскадна зміна однакових папок** (ТЗ 21.1): якщо змінити назву папки, що
  зустрічається в інших шляхах, програма пропонує «Так, до всіх / Ні, лише цей /
  Скасувати» й оновлює всі відповідні шляхи.
- **Детектор конфліктів** (`Conflicts.cs`): дублікати імен, недопустимі символи,
  зарезервовані імена Windows (CON/PRN/NUL…), порожні імена, наявні на диску
  об'єкти, перевищення бюджету. Конфлікти показуються в колонці «Конфлікт»;
  **критичні блокують застосування** до виправлення.
- **Менеджер словника** (`DictionaryDialog`): перегляд, додавання, редагування,
  видалення записів; **затвердження LLM-скорочень** (llm→manual) — лише після
  цього їх бачить звичайний аналіз. Ручні записи мають пріоритет.
- **Журнал виконання** (`ExecutionLog` → `execution.log`): сканування, аналіз,
  провайдер, лічильники пакетів, застосування, конфлікти, зупинка, помилки —
  **без API-ключів**.
- Колонка «Джерело» у плані: без змін / словник+алгоритм / словник+ШІ / ручне.

## v1.7.0 — провайдери ШІ

- **П'ять провайдерів інтелектуального аналізу**: Ollama (локальна й віддалена),
  Claude, OpenAI, Grok, Gemini — єдиний HTTP-клієнт із «діалектами»
  (`Ai/AiChatModel.cs`); OpenAI і Grok сумісні за протоколом.
- **Профілі з незалежним зберіганням ключів**: для кожного провайдера — кілька
  профілів (назва, ключ, baseUrl, модель, статус, примітка); перемикання провайдера
  не зачіпає чужі ключі. API-ключі **шифруються Windows DPAPI**, у відкритому
  вигляді не зберігаються (`Ai/AiProfiles.cs`). Додати/редагувати/видалити/зробити
  активним/перевірити — у блоці «Провайдер» вкладки «Налаштування».
- **Попередження**: платний API (Claude/OpenAI/Grok/Gemini) — підтвердження перед
  запуском; віддалена Ollama — нейтральне сповіщення; локальна — без попередження.
- **Пакетна обробка** (`Ai/BatchAnalyzer.cs`): до моделі йдуть лише УНІКАЛЬНІ
  сегменти, що реально перевищують ліміти; повторювані частини шляхів групуються
  (один запит), ПІБ і словникові — не надсилаються; дати відрізаються до моделі.
- **Структурований JSON + локальна валідація** (ТЗ 9.3/9.4): відповідь моделі
  проходить перевірку (валідний JSON, ПІБ, недопустимі символи, «не збільшує
  довжину», акронім лише як справжні ініціали слів); невалідне не застосовується.
- LLM-скорочення кешуються окремо (`source='llm'`) і **не потрапляють** у
  затверджений словник — звичайний аналіз їх не бачить.

## v1.6.0 — правила скорочення

- **ПІБ недоторканні**: сегменти з прізвищами та ініціалами («ЛЕВЧУК_О_О…»,
  «Чаричанський П.О.») виявляються кодом і не скорочуються взагалі — модель їх
  навіть не бачить (`SafeShortening.cs`).
- **Валідація скорочень**: порожні, ті, що не зменшують довжину, та «сирі»
  акроніми від моделі відхиляються (оригінал повертається без змін);
  затверджені словникові абревіатури (КСУ, ВРП…) працюють як раніше.
- **Не скорочувати без потреби**: підтверджено тестами — для назв у межах
  лімітів модель/правила не викликаються жодного разу.
- **Окремі ліміти довжини** імені файлу та імені папки (на додачу до бюджету
  повного шляху), 0 — вимкнено; дати й розширення не страждають.
- **Налаштовувані правила звичайного аналізу** (блок у «Налаштуваннях»):
  словник типових слів, службові слова, об'єднання розділювачів, видалення
  дубльованих слів, нормалізація дат («13 червня 2025» → «13.06.2025»),
  акроніми зі слів (вимкн. за замовч.), пословне обрізання до N символів
  (вимкн., крайній засіб — застосовується лише коли інші правила не допомогли).
- Колонка **«Джерело»** у таблиці плану: без змін / словник+алгоритм / словник+ШІ.

## v1.5.0 — фундамент UI

- Правий сайдбар із вкладками **Папки / Архіви / Налаштування**; джерела можна
  вмикати/вимикати чекбоксом перед аналізом.
- **Дві мови інтерфейсу**: українська (за замовчуванням) та англійська; усі тексти
  винесено в ресурси `Resources/Localization/uk-UA.json`, `en-US.json`; мова
  зберігається між запусками.
- **Вибір теми**: світла (за замовчуванням), темна (м'яка графітова, не чорна),
  системна; тема зберігається між запусками і застосовується до всіх вікон.
- Кнопка **«Проаналізувати»** — звичайний аналіз без ШІ (словник + правила);
  кнопка **«Інтелектуальний аналіз»** — аналіз через модель Ollama.
- Кнопка **«Зупинити»** біля прогрес-бару: коректне скасування між елементами,
  без аварійного завершення.
- Розділ **«Про програму»**: автор, контакти, ліцензія MIT, перелік можливостей.
- Алгоритмічні та LLM-скорочення більше не потрапляють у затверджений словник
  автоматично (звичайний аналіз використовує лише записи `dict`/`manual`).

## v1.4.0 — Ollama-панель і англійські імена файлів

- Панель Ollama: статус сервера, вибір моделі з рекомендованих і встановлених,

## v1.3.0 — Avalonia, компактність, швидкість

- Інтерфейс перенесено з WinForms на **Avalonia 11** (сучасна світла тема,
  картки, акцентні кнопки).
- Кеш патернів: DuckDB → **JSON-файл** (−35 МБ, читається вручну).
- Розпакування архіву **одним викликом 7z** замість процесу на файл (у рази швидше).
- Санітизація імен: кінцеві крапки/пробіли та заборонені символи Windows
  вичищаються з планів (виправлено розходження плану з диском).
- Дві збірки: compact (~26 МБ, потрібен .NET 8) і portable (~44 МБ, самодостатня).

## v1.2.0 — захист дат

- Дати в назвах (`21.04.2021`, `09_01_2025`, `2025-01-09`, `01.2025`,
  «13 червня 2025», рік) вирізаються ДО виклику моделі й повертаються після;
  жорстке обрізання ніколи не зрізає дату.

## v1.1.0 — папки як джерело

- Аналіз не лише архівів, а й **папок з диска**: перейменування на місці

  теку зі скороченими іменами.
- Індивідуальна ціль для кожного джерела; прогрес-бар; журнал CSV до застосування.

## v1.0.0 — базовий випуск
